## 1. What is Networking?

### 📘 Formal Definition

**Computer networking** is the practice of connecting two or more computing devices together using physical or wireless links so that they can **exchange data, share resources, and communicate** with one another using agreed-upon rules called **protocols**.

### 🧠 Simple Explanation

Strip away the jargon, and networking is really just one idea repeated everywhere:

> **Getting information from Point A to Point B, reliably and correctly.**

A "network" is simply a group of devices — computers, phones, printers, servers, smart TVs — that are wired or wirelessly linked so they can talk to each other.

### 🏙️ Real-World Analogy: The Postal System

Think of networking like a **city's postal and road system**:

| Postal System Concept | Networking Equivalent |
|---|---|
| Houses and buildings | Devices (computers, phones, servers) |
| Street addresses | IP Addresses *(Chapter 3)* |
| Postal codes / sorting hubs | Routers *(Chapter 13)* |
| Roads and highways | Network cables / wireless signals (media) |
| Mail trucks | Data packets |
| Language on the letter | Protocols (e.g., TCP/IP) |
| Post office sorting rules | Routing rules |

Just like a letter needs a correct address, a proper format, and a delivery path to reach its destination, **data on a network needs an address, a structure, and a path** to reach the correct device.

### 🌍 Real-World Examples

- Sending a WhatsApp message to a friend
- Streaming a movie on Netflix
- A laptop printing a document to a wireless printer
- An employee accessing a company file server from home
- A smart thermostat reporting data to a mobile app
- An ATM communicating with a bank's central server

Every single one of these relies on networking happening silently in the background.

## 2. Why Networking Exists

Networking didn't appear for one single reason — it evolved because isolated computers had **real limitations**. Below are the core motivations.

### 🔁 Resource Sharing

Before networks, every computer needed its own printer, storage, and software. Networking allows:
- Shared printers, scanners, and storage drives
- Shared internet connections
- Shared applications and databases

**Example:** An office of 50 employees uses **one** networked printer instead of 50 individual printers.

### 💬 Communication

Networks enable instant communication regardless of physical distance:
- Email, instant messaging, video calls
- File transfers
- Real-time collaboration tools (Google Docs, Slack, Teams)

### 🌐 Internet Access

The Internet is simply the **largest network of networks** in existence. Without local networking concepts (addressing, routing, transmission), the Internet itself could not function.

### ☁️ Cloud Computing

Cloud computing — storing and processing data on remote servers instead of your own device — is **entirely dependent on networking**. When you upload a file to Google Drive or run an app on AWS, your device is constantly communicating over a network with remote data centers.

### 🏗️ Modern Infrastructure

Modern society runs on networked systems:
- Banking and financial transactions
- Hospitals and medical record systems
- Power grids and industrial control systems (SCADA/ICS)
- Transportation systems (traffic lights, GPS, flight control)
- Government and military communications

> 🔑 **Core idea:** Networking exists because **isolated systems are limited**, and **connected systems are powerful** — but that power comes with responsibility, which is where cybersecurity comes in.

## 3. Why Networking Matters in Cybersecurity

Networking is the **foundation** of cybersecurity. You cannot defend or attack what you do not understand.

### 🎯 Why Attackers Need Networking Knowledge

Attackers exploit networks to:
- Scan for live hosts and open ports (reconnaissance)
- Intercept unencrypted traffic (sniffing, man-in-the-middle attacks)
- Exploit misconfigured devices (routers, firewalls, IoT devices)
- Move laterally across a network after gaining initial access
- Launch denial-of-service (DoS) attacks by overwhelming network resources
- Spoof addresses to impersonate trusted devices

Without networking knowledge, an attacker cannot understand **how to reach a target, how data flows, or where weaknesses exist.**

### 🛡️ Why Defenders Need Networking Knowledge

Defenders (Blue Team, SOC analysts, network admins) need networking to:
- Design secure network architectures (segmentation, firewalls, VLANs)
- Monitor traffic for anomalies (using tools like Wireshark — *Chapter 16*)
- Detect intrusions (IDS/IPS systems)
- Configure access control and routing securely
- Investigate incidents by tracing how an attacker moved through the network

### 🔗 The Relationship Between Networking and Security

| Without Networking Knowledge | With Networking Knowledge |
|---|---|
| Security feels like "black box" rules | Security decisions are based on real traffic behavior |
| Can't read packet captures or logs | Can analyze and trace attacks |
| Can't configure firewalls properly | Can build layered, defensible architecture |
| Can't understand attack techniques | Can anticipate and prevent them |

> 🔑 **Golden Rule of Cybersecurity:** *"You cannot secure what you do not understand."* Networking fundamentals are the prerequisite to almost every security skill — from penetration testing (Nmap, *Chapter 17*) to defense (*Chapter 19*).

## 4. History and Evolution of Networking

Understanding *why* networking looks the way it does today requires a quick look at *how* it got here.

### 🕰️ Early Computer Communication (1950s–1960s)

- Early computers were **massive, standalone machines** with no way to communicate with each other.
- Data was physically transported between machines using **punch cards** and **magnetic tapes**.
- The concept of "remote" computer access began with **time-sharing systems**, where multiple terminals connected to one central mainframe.

### 🚀 ARPANET (1969)

- **ARPANET** (Advanced Research Projects Agency Network), funded by the U.S. Department of Defense, is widely considered the **first operational packet-switching network** — the ancestor of the modern Internet.
- It connected four university research computers: UCLA, Stanford, UC Santa Barbara, and the University of Utah.
- It introduced the foundational idea of **packet switching** — breaking data into small chunks ("packets") that travel independently and get reassembled at the destination, instead of relying on a single dedicated line for the entire message.
- This was a revolutionary shift from older **circuit-switched** systems (like traditional telephone networks), where a dedicated line was reserved for an entire conversation.

### 🌐 The Birth of TCP/IP and the Modern Internet (1970s–1990s)

- In the 1970s, **Vint Cerf** and **Bob Kahn** developed **TCP/IP**, the protocol suite that standardized how data is addressed, transmitted, and routed.
- In 1983, ARPANET officially adopted TCP/IP — this date is often considered the "birth of the modern Internet."
- The 1990s brought the **World Wide Web** (invented by Tim Berners-Lee in 1989–1991), which made the Internet usable for the general public through web browsers.
- Commercial Internet Service Providers (ISPs) emerged, bringing internet access to homes and businesses.

### 📱 Why Networking Became Important

| Era | Driving Force |
|---|---|
| 1960s–70s | Military and academic research collaboration |
| 1980s–90s | Business connectivity, email, early internet |
| 2000s | Broadband, social media, e-commerce explosion |
| 2010s | Mobile devices, cloud computing, IoT |
| 2020s+ | 5G, edge computing, massive cyber-threat landscape |

Today, networking isn't optional infrastructure — it's the **backbone of the global economy, communication, and digital security.**

## 5. Types of Networks

Networks are classified primarily by **geographic range** and **purpose**.

### 📍 PAN — Personal Area Network

| Attribute | Details |
|---|---|
| **Definition** | A network for personal devices within a very short range, typically centered around one individual |
| **Range** | ~1–10 meters |
| **Examples** | Bluetooth headphones connected to a phone, smartwatch syncing with a phone |
| **Advantages** | Very low power use, simple setup, no infrastructure needed |
| **Disadvantages** | Very limited range, low data capacity, device limits |
| **Real-World Usage** | Wireless earbuds, fitness trackers, wireless keyboards/mice |

### 🏠 LAN — Local Area Network

| Attribute | Details |
|---|---|
| **Definition** | A network confined to a small physical area such as a home, office, or single building |
| **Range** | A single building or campus (up to ~1 km) |
| **Examples** | Home Wi-Fi network, office network, school computer lab |
| **Advantages** | High speed, low latency, easy to manage, cost-effective |
| **Disadvantages** | Limited geographic coverage |
| **Real-World Usage** | Connecting computers, printers, and servers in an office |

### 📶 WLAN — Wireless Local Area Network

| Attribute | Details |
|---|---|
| **Definition** | A LAN that uses wireless signals (Wi-Fi) instead of physical cables |
| **Range** | Similar to LAN, typically 30–100 meters indoors |
| **Examples** | Home Wi-Fi, coffee shop Wi-Fi, office wireless network |
| **Advantages** | No cabling needed, mobility, easy device addition |
| **Disadvantages** | Susceptible to interference, generally less secure than wired, can be slower/less stable than wired LAN |
| **Real-World Usage** | Laptops and phones connecting to "Wi-Fi" at home, school, or work |

### 🏢 CAN — Campus Area Network

| Attribute | Details |
|---|---|
| **Definition** | A network that spans multiple buildings within a limited geographic area, like a university or corporate campus |
| **Range** | Several buildings, up to a few kilometers |
| **Examples** | A university connecting its library, dorms, and admin buildings |
| **Advantages** | Centralized management across multiple buildings, faster than WAN |
| **Disadvantages** | More expensive than a single LAN, requires more infrastructure |
| **Real-World Usage** | University campuses, hospital campuses, corporate headquarters with multiple buildings |

### 🏙️ MAN — Metropolitan Area Network

| Attribute | Details |
|---|---|
| **Definition** | A network that spans a city or large metropolitan area |
| **Range** | A city or large town (up to ~50 km) |
| **Examples** | City-wide public Wi-Fi, cable TV networks, municipal government networks |
| **Advantages** | Connects multiple LANs across a wide urban area, faster than WAN |
| **Disadvantages** | Expensive to build and maintain, harder to secure due to scale |
| **Real-World Usage** | ISPs providing city-wide internet, metropolitan government services |

### 🌍 WAN — Wide Area Network

| Attribute | Details |
|---|---|
| **Definition** | A network that spans large geographic areas — even countries or continents |
| **Range** | Countries, continents, global |
| **Examples** | The Internet itself, a multinational company's network connecting offices worldwide |
| **Advantages** | Connects geographically distant locations, enables global communication |
| **Disadvantages** | Higher latency, more expensive, more complex to secure and manage |
| **Real-World Usage** | The Internet, banks connecting branches across countries, multinational corporate VPNs |

### 📊 Comparison Table: All Network Types

| Network Type | Full Name | Typical Range | Example |
|---|---|---|---|
| **PAN** | Personal Area Network | ~1–10 m | Bluetooth earbuds |
| **LAN** | Local Area Network | One building | Home/office network |
| **WLAN** | Wireless LAN | One building (wireless) | Home Wi-Fi |
| **CAN** | Campus Area Network | Multiple buildings | University campus |
| **MAN** | Metropolitan Area Network | A city | City-wide ISP network |
| **WAN** | Wide Area Network | Countries/global | The Internet |

```
PAN  <  LAN/WLAN  <  CAN  <  MAN  <  WAN
(small)                              (large)
```

## 6. Network Architectures

A **network architecture** describes how devices on a network interact and share responsibilities.

### 🖥️ Client-Server Architecture

```
   [ Client 1 ]----\
   [ Client 2 ]-----\         REQUEST  --->
   [ Client 3 ]------>------[ SERVER ]
                            <---  RESPONSE
```

**Components:**
- **Client** — a device that requests services or data (e.g., your web browser)
- **Server** — a powerful, centralized machine that provides services or data (e.g., a web server hosting a website)

**Workflow:**
1. The client sends a **request** to the server (e.g., "Give me this webpage").
2. The server processes the request.
3. The server sends a **response** back to the client (e.g., the webpage content).

**Advantages:**
- Centralized control, easier to manage and secure
- Centralized data backup
- Scalable for many users

**Disadvantages:**
- Server is a **single point of failure** — if it goes down, all clients are affected
- Can become expensive to scale (more powerful servers needed)
- Server can become a bottleneck under heavy load

**Examples:** Web browsing (browser = client, website = server), email systems, online banking.

### 🔄 Peer-to-Peer (P2P) Architecture

```
   [ Peer A ] <-----> [ Peer B ]
        ^                  ^
        |                  |
        v                  v
   [ Peer D ] <-----> [ Peer C ]
```

**Components:**
- **Peers** — devices that act as *both* clients and servers, sharing resources directly with one another, with no central authority

**Workflow:**
1. Each device (peer) can request resources from, and provide resources to, any other peer.
2. There is no central server controlling the network.

**Advantages:**
- No single point of failure
- Cost-effective (no dedicated server required)
- Scales well for direct file sharing

**Disadvantages:**
- Harder to secure and manage (no central control)
- Performance depends on individual peers
- Difficult to maintain consistency and data integrity across all peers

**Examples:** BitTorrent file sharing, some early instant messaging systems, blockchain networks.

### ⚖️ Client-Server vs. Peer-to-Peer

| Feature | Client-Server | Peer-to-Peer |
|---|---|---|
| Control | Centralized | Decentralized |
| Single point of failure | Yes (the server) | No |
| Setup cost | Higher (dedicated server) | Lower |
| Security management | Easier (centralized) | Harder (distributed) |
| Best for | Businesses, websites, banking | File sharing, small networks |

## 7. Network Topologies

A **topology** is the arrangement/layout of devices and connections in a network.

### 🚌 Bus Topology

```
 [PC1]---[PC2]---[PC3]---[PC4]
     ===== Single Backbone Cable =====
```

- **Advantages:** Simple, inexpensive, easy to set up for small networks
- **Disadvantages:** If the main cable fails, the *entire* network goes down; performance degrades as more devices are added
- **Modern Usage:** Largely obsolete; rarely used in modern networks

### ⭐ Star Topology

```
            [ Switch/Hub ]
           /     |      \
      [PC1]   [PC2]   [PC3]
```

- **Advantages:** If one cable/device fails, only that device is affected; easy to add/remove devices; easy to troubleshoot
- **Disadvantages:** If the central device (switch) fails, the whole network goes down
- **Modern Usage:** **The most common topology today** — used in virtually all homes and offices

### ⭕ Ring Topology

```
   [PC1]---[PC2]
     |        |
   [PC4]---[PC3]
```

- **Advantages:** Data flows in a predictable, orderly manner; performs consistently under heavy load
- **Disadvantages:** A single broken connection can disrupt the entire ring (unless dual-ring)
- **Modern Usage:** Rare today; some legacy and specialized industrial networks still use it

### 🕸️ Mesh Topology

```
   [PC1]------[PC2]
     | \      / |
     |  \    /  |
     |   \  /   |
   [PC4]------[PC3]
```

- **Advantages:** Highly redundant and fault-tolerant — multiple paths between devices; excellent reliability
- **Disadvantages:** Expensive and complex to set up due to large amount of cabling/connections required
- **Modern Usage:** Used in mission-critical networks, mesh Wi-Fi systems, and military/backbone networks

### 🔀 Hybrid Topology

```
   [Star Group A]----[Mesh Backbone]----[Star Group B]
```

- **Advantages:** Combines strengths of multiple topologies, flexible and scalable
- **Disadvantages:** Can be complex and costly to design and manage
- **Modern Usage:** Common in large enterprise networks and ISP infrastructures

### 📊 Topology Comparison Table

| Topology | Reliability | Cost | Scalability | Modern Usage |
|---|---|---|---|---|
| Bus | Low | Very Low | Poor | Obsolete |
| Star | Medium-High | Low-Medium | Good | **Very Common** |
| Ring | Medium | Medium | Medium | Rare |
| Mesh | Very High | High | Excellent | Critical infrastructure, mesh Wi-Fi |
| Hybrid | High | Varies | Excellent | Large enterprises, ISPs |

## 8. Basic Network Components

A brief, high-level introduction. Deeper technical detail will follow in later chapters.

| Device | What It Does (Simple Explanation) |
|---|---|
| **Router** | Connects different networks together and directs data between them (e.g., your home network to the Internet). Covered in depth in *Chapter 14*. |
| **Switch** | Connects multiple devices within the *same* network and forwards data only to the intended device. Covered in *Chapter 15*. |
| **Hub** | An older, simpler device that connects multiple devices but broadcasts data to *all* of them (mostly obsolete, replaced by switches). |
| **Access Point (AP)** | Allows wireless devices to connect to a wired network — essentially a "bridge" between Wi-Fi and the LAN. |
| **Modem** | Converts signals from your ISP (e.g., cable or fiber signals) into a format your network/router can use, and vice versa. |
| **Firewall** | Monitors and controls incoming/outgoing traffic based on security rules, acting as a barrier between trusted and untrusted networks. |
| **IDS** (Intrusion Detection System) | Monitors network traffic for suspicious activity and **alerts** administrators — it does not block traffic. |
| **IPS** (Intrusion Prevention System) | Like an IDS, but can **actively block or stop** detected threats in real time. |

> These devices will each receive a dedicated, in-depth chapter later in this knowledge base.

## 9. Network Media

"Media" refers to the physical or wireless pathway data travels through.

| Media Type | Advantages | Disadvantages | Common Uses |
|---|---|---|---|
| **Ethernet Cable (general)** | Reliable, fast, relatively low cost | Limited by cable length, requires physical installation | Wired LANs |
| **Cat5e** | Affordable, supports up to 1 Gbps | Lower max speed/distance than newer standards | Home and small office networks |
| **Cat6** | Supports higher speeds (up to 10 Gbps over shorter distances), better noise resistance | More expensive than Cat5e, thicker/less flexible cable | Modern offices, data centers |
| **Coaxial Cable** | Good shielding against interference, long-standing reliability | Bulkier, largely replaced by fiber/twisted-pair for new installs | Cable TV/Internet (older ISP infrastructure) |
| **Fiber Optic Cable** | Extremely high speed, very long distances, immune to electromagnetic interference | Expensive, fragile, requires specialized equipment to install/repair | Backbone Internet infrastructure, high-speed ISPs, data centers |
| **Wireless Communication** | No cables, high mobility, easy to extend coverage | Susceptible to interference and signal loss, generally less secure by default | Wi-Fi, mobile networks, IoT devices |

## 10. Wireless Networking Basics

A high-level overview — wireless protocols will not be explored in deep technical detail here.

| Technology | What It Is |
|---|---|
| **Wi-Fi** | A wireless technology that allows devices to connect to a LAN/Internet without cables, using radio waves |
| **WLAN** | The network created when devices connect via Wi-Fi (already covered in Section 5) |

## 11. Data Transmission Fundamentals

Understanding *how well* data moves is just as important as understanding *that* it moves.

| Concept | Definition | Real-World Example | Real-World Impact |
|---|---|---|---|
| **Bandwidth** | The *maximum* amount of data that can theoretically be transmitted over a connection in a given time (measured in Mbps/Gbps) | A 100 Mbps internet plan | Higher bandwidth allows more data to flow, but doesn't guarantee actual speed |
| **Throughput** | The *actual* amount of data successfully transmitted in real-world conditions | Your real download speed during a speed test | Often lower than advertised bandwidth due to congestion, distance, interference |
| **Latency** | The delay (time) it takes for data to travel from source to destination | "Ping" time in online gaming | High latency causes lag, even with high bandwidth |
| **Jitter** | The variation in latency over time (inconsistent delay) | Choppy video calls | Causes stuttering in video/audio streaming and VoIP calls |
| **Packet Loss** | When data packets fail to reach their destination | Pixelated/frozen video during a call | Causes broken audio, missing data, and connection drops |

### Quick Memory Trick

- High Bandwidth + Low Latency = Ideal
- High Bandwidth + High Latency = Large but delayed transfers
- Low Bandwidth + Low Latency = Responsive but limited capacity
- High Jitter = Poor voice/video call quality
- Packet Loss = Missing data and connection instability

### 🧠 Analogy: Bandwidth vs. Throughput

> Imagine a highway with **8 lanes (bandwidth)**. If there's heavy traffic, even though 8 lanes exist, cars (data) might only be able to move through at a rate equal to 3 lanes worth of capacity — that's your **throughput**.

## 12. Network Interface Cards (NICs)

### 🎯 Purpose

A **Network Interface Card (NIC)** is the hardware component that allows a device to physically connect to a network. Every device that communicates on a network — laptops, desktops, servers, phones — has one (built-in or external).

### 🔌 Wired NIC

- Connects via an Ethernet cable
- Generally offers more stable and faster connections
- Common in desktops, servers, and gaming consoles

### 📡 Wireless NIC

- Connects via radio waves (Wi-Fi)
- Offers mobility and flexibility
- Common in laptops, smartphones, and tablets

### ⭐ Importance

Without a NIC, a device **physically cannot** participate in a network — it has no way to send or receive network signals. Every NIC also has a **unique hardware identifier called a MAC Address**, which will be covered in depth in *Chapter 6*.

> 🔑 MAC Addresses are covered in depth in **Chapter 4: MAC Addresses** and are one of the most important concepts for understanding local network communication.

## 13. How Data Travels Across a Network

Here's a simplified, beginner-friendly path data takes when you do something like load a website. (Protocol-level detail is intentionally left for later chapters.)

```
 [ You / User ]
        |
        v
 [ Your Device ] -- (e.g., laptop/phone)
        |
        v
 [ Home Router ] -- (directs traffic in/out of your home network)
        |
        v
 [ ISP (Internet Service Provider) ] -- (connects you to the wider Internet)
        |
        v
 [ The Internet ] -- (a global network of networks/routers)
        |
        v
 [ Destination Server ] -- (e.g., the website's server)
        |
        v
 [ Response travels back the same path to your device ]
```

### Step-by-Step (Simplified)

1. **User** initiates an action — e.g., typing a website address into a browser.
2. The **device** prepares the request and sends it out through its NIC.
3. The **router** receives the request and forwards it toward the Internet.
4. The **ISP** routes the traffic onto the broader Internet backbone.
5. The request travels across multiple interconnected networks/routers until it reaches the **destination server**.
6. The server processes the request and sends a **response** back along a similar path.
7. Your device receives the response and displays it (e.g., renders the webpage).

> This entire process typically happens in **milliseconds**. The "how" behind each step — addressing, routing protocols, packet structure — is covered extensively in upcoming chapters (IP Addressing, Routing, TCP/IP Model, etc.).

| **Broadcast** | Communication sent to all devices on a local network |

## 15. Common Beginner Misconceptions

| ❌ Misconception | ✅ Reality |
|---|---|
| "Internet" and "Wi-Fi" are the same thing | Wi-Fi is just *one method* of connecting to a network; the Internet is the global network itself. You can have Wi-Fi without Internet (e.g., a local-only wireless network) |
| Bandwidth and speed are the same thing | Bandwidth is the *maximum capacity*; actual speed (throughput) depends on real-world conditions like congestion and interference |
| More Wi-Fi signal bars = faster Internet | Signal bars indicate *signal strength*, not speed. A strong signal to a congested or slow connection can still be slow |
| A faster router always means faster Internet | Your speed is also limited by your ISP plan and the capabilities of connected devices |
| Private browsing/incognito mode hides your activity from your ISP or network | It only prevents *local* browser history from being saved — your ISP and network admin can still see your traffic |
| Having a firewall means you're "fully protected" | A firewall is just *one layer* of security — defense requires multiple layers (covered in *Chapter 19: Network Defense*) |
| All wireless networks are insecure by nature | Security depends on configuration (encryption, authentication) — not just whether it's wired or wireless |
| Networking is only important for network engineers | Networking knowledge is required in cybersecurity, cloud computing, system administration, DevOps, and many other IT fields |

## 16. Industry Perspective

How networking concepts manifest differently across real-world environments:

### 🏠 Home Networks
- Typically a single router/modem combo
- One Wi-Fi network (sometimes a guest network)
- Few connected devices (phones, laptops, smart TVs, IoT devices)
- Minimal formal security beyond a Wi-Fi password and basic firewall

### 🏢 Office Networks
- Multiple switches and access points
- Segmented networks (e.g., separate networks for guests vs. employees)
- Centralized management via IT department
- Firewalls, VPNs for remote access, and monitoring tools in place

### 🖥️ Data Centers
- Massive numbers of servers, switches, and routers
- High-speed fiber optic connections
- Redundant power and network paths for reliability (high availability)
- Strict physical and digital security controls

### ☁️ Cloud Environments
- Virtualized networking (software-defined networks)
- Resources spread across multiple physical data centers globally
- Scalable on-demand — networks can expand/shrink based on need
- Security managed through cloud-specific tools (security groups, virtual firewalls, identity management)

| Environment | Scale | Complexity | Primary Concern |
|---|---|---|---|
| Home | Very Small | Low | Convenience, basic security |
| Office | Small–Medium | Medium | Productivity, internal security |
| Data Center | Large | High | Reliability, performance, physical security |
| Cloud | Massive/Global | Very High | Scalability, virtualized security |

## 17. Key Takeaways

### 📋 Summary

- Networking is the practice of connecting devices to share data and resources using protocols.
- Networking exists to enable resource sharing, communication, Internet access, and modern infrastructure like the cloud.
- Networking knowledge is foundational to **both offensive and defensive cybersecurity**.
- Networking evolved from early point-to-point systems to ARPANET, and eventually to the modern TCP/IP-based Internet.
- Networks are classified by range: **PAN → LAN/WLAN → CAN → MAN → WAN**.
- Two primary architectures exist: **Client-Server** (centralized) and **Peer-to-Peer** (decentralized).
- Topologies (**Bus, Star, Ring, Mesh, Hybrid**) define how devices are physically/logically arranged.
- Basic devices like routers, switches, and firewalls each play a distinct role.
- Network media (cables and wireless) and wireless technologies (Wi-Fi, Bluetooth, cellular) form the physical/wireless backbone.
- Performance is measured through bandwidth, throughput, latency, jitter, and packet loss.
- Every networked device requires a NIC to participate in a network.
- Data travels through a predictable path: **User → Device → Router → ISP → Internet → Server**.
- Every advanced networking and cybersecurity topic ultimately builds upon the concepts introduced in this chapter.
